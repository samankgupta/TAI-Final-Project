2. RANKING PIPELINE (5 STAGES)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Stage 1: Filter by Job Role
  └─ Input: All 10,174 resumes
  └─ Output: E-commerce Specialist subset (11 resumes in eval sample)
  
Stage 2: TF-IDF Ranking (Keyword Matching)
  └─ Method: Term frequency-inverse document frequency
  └─ Accuracy: 40% (top-5)
  
Stage 3: Embedding Ranking (Semantic Similarity)  
  └─ Model: all-MiniLM-L6-v2
  └─ Accuracy: 60% (top-5) ← BEST SINGLE METHOD
  
Stage 4: Gemini Ranking (LLM-Based)
  └─ API: Google Generative AI with rate limiting
  └─ Rate Limit: 13 requests/minute (enforced)
  └─ Accuracy: 40% (mock scoring in test)
  
Stage 5: Ensemble Ranking (Improvement)
  └─ Method: Weighted aggregation with confidence scoring
  └─ Accuracy: 40% (FAILED - worse than embeddings)
  └─ Status: ✗ REGRESSION from 60% to 40%

   └─ Semantic similarity (60%) beats TF-IDF (40%)
   └─ Implication: Context matters more than exact keywords
   
2. HIGH METHOD AGREEMENT (90% TSS)
   └─ Different methods largely agree on top candidates
   └─ Implication: Use multiple methods for confidence scoring
   
3. LLM DOESN'T ADD VALUE YET
   └─ Gemini mock scoring gave uniform results (all 5/10)
   └─ Implication: Need real hallucination detection to add signal
   
4. ENSEMBLE AGGREGATION FAILED
   └─ Simple weighting reduced accuracy from 60% to 40%
   └─ Implication: Selective prediction > forced averaging
   
5. RATE LIMITING WORKS PERFECTLY
   └─ Proper enforcement with no violations
   └─ Implication: System respects API constraints
